# Contributing Feedback

Thank you for helping improve SaaSy Platform.

## Before opening an issue

1. Search existing open and closed issues.
2. Choose the most appropriate issue form.
3. Submit one problem or request per issue.
4. Remove sensitive or customer-identifying information from screenshots and logs.

## Effective bug reports

Include the affected page or feature, steps to reproduce, expected behavior, actual behavior, environment details, frequency, and safe supporting evidence. Correlation or request IDs are helpful when they do not reveal secrets.

## Feature requests

Describe the problem and desired outcome before proposing a specific implementation. Explain who benefits and how the improvement would be used.

## Public communication

Be respectful and constructive. Submission does not guarantee acceptance, prioritization, or a release date. Maintainers may edit titles, request more information, close duplicates, or move accepted work to a private backlog.

## Security and privacy

Never disclose credentials, tokens, private tenant data, personal data, or vulnerability details in a public issue. Use the process in SECURITY.md for suspected security vulnerabilities.
